# Renewl MVP — Implementation Plan

## Context

Renewl ships its first usable version on `dev/mvp1` → Vercel project `renewls-dev`. The live waitlist on `main` → `renewl-live.vercel.app` (Vercel project `renewl-live`) stays untouched.

**What we're delivering** (per `docs/scope.md`):
- magic-link sign-in via Convex Auth + Resend
- two ingestion paths (PDF/image upload + paste-text), parsed by OpenAI `gpt-4o-mini` via Vercel AI SDK in a Convex Node action
- editable parse-review card → confirms into a 10-item subscriptions tracker
- 10-cap replacement flow via `pendingParses`
- manual entry path (cap-aware hard-block)
- daily 08:00 IST renewal alerts via Resend (durable via `@convex-dev/resend` component)
- 24h file cleanup cron
- existing waitlist landing remains at `/`, untouched

**Locked decisions (this session):**
- AI provider: direct via `@ai-sdk/openai` (model `gpt-4o-mini`) — switched from `@ai-sdk/anthropic` on 2026-04-25 to use existing OpenAI credits
- Email sender: real verified domain (e.g. `renewl.in`/`getrenewl.com`) — user buys + verifies before push; sandbox `onboarding@resend.dev` only as interim
- Cadence: all slices in one push to `dev/mvp1`

**Already done:**
- Convex schema written + pushed (`convex/schema.ts` → deployment `kindly-quail-882`)
- Vercel `renewls-dev` linked, env vars `NEXT_PUBLIC_CONVEX_URL` + `NEXT_PUBLIC_CONVEX_SITE_URL` set
- Memory: deploys via git-push to GitHub (no `vercel deploy` from CLI); Convex pushes via `npx convex dev` are fine

## §0. Pre-flight (human-only, one-time)

2. Resend dashboard → Domains → Add → enter apex.

## Slice C — Parser action (parallel)

**Packages:** `ai`, `@ai-sdk/openai`, `zod`.

**File to create:** `convex/parser.ts` — top of file: `"use node";`
- `internalAction run({ jobId })`:
  1. `markRunning` → flips status, ++attempts.
  2. `loadJob` → returns row.
  3. If `fileId`: `ctx.storage.get(fileId)` + read mime via `ctx.db.system.get("_storage", fileId)`. Image MIME → `type: "image"` block. PDF → `type: "file"` block with `mediaType: "application/pdf"`. OpenAI's chat-completions API has native PDF + image input via the AI SDK's file/image blocks; no OCR needed.
  4. If `pastedText`: `type: "text"` block.
  5. Call AI SDK `generateObject` (verified present in v6 from `node_modules/ai/dist/index.d.ts`; lints that flag it as removed are stale). Model: `openai("gpt-4o-mini")` from `@ai-sdk/openai`.
  6. Validate via Zod schema mirroring `extractedValidator`.
  7. Coerce: ISO date → `toStartOfDayIstMs(iso)`; rupees float → `Math.round(rupees * 100)` paise.
  8. `markSucceeded` patches `extracted` + status.
- `internalQuery loadJob`, `internalMutation markRunning|markSucceeded|markFailed`.

**File to create:** `convex/lib/dates.ts` — `toStartOfDayIstMs`, `startOfTomorrowIstMs`. Uses `Intl.DateTimeFormat("en-IN", { timeZone: "Asia/Kolkata" })` + `Date.UTC`. No libraries.

**Prompt** (system message, calibrated for Indian receipts):
- Role + scope ("extract subscription data from Indian consumer receipts")
- Common Indian vendors per category (bias toward recognizable strings)
- Currency: always INR; convert USD at 84 INR/USD
- Date: `nextRenewal` as `YYYY-MM-DD`; one-time → 30 days from today
- Cycle: enum reminder; uncertain → monthly
- Category: enum slugs only; unfit → `other`
- isSubscription: only true for recurring services (not food, single purchases, ride-hailing)
- Confidence calibration: 0.9+ explicit, 0.6–0.9 inferred, <0.6 guessing

**Zod schema** (parser-friendly shapes, coerced after validation):
- `vendor: z.string().min(1)`, `amountInr: z.number().positive()` (rupees), `cycle: z.enum([...])`, `nextRenewal: z.string().regex(/^\d{4}-\d{2}-\d{2}$/)`, `category: z.enum([... 11 slugs])`, `isSubscription: z.boolean()`, `confidence: z.number().min(0).max(1)`.

**Errors:** Catch `NoObjectGeneratedError`. `lastError = err.message + " | " + err.text`. No auto-retry; UI shows manual retry. Other errors: `lastError = String(err)`.

**Risks:**
- `"use node";` in `parser.ts` only. `parseJobs.ts` (mutations) must stay in default runtime.
- 10MB upload cap keeps us under OpenAI's chat-completions file size limit.
- Cost on `gpt-4o-mini`: ~$0.15/1M input + $0.60/1M output → roughly ₹0.05–0.15 per receipt parse including image tokens.

**Verification:** Real Netflix India PDF → succeeded in <5s with vendor "Netflix", amountInr ~64900, cycle monthly, category streaming, confidence ≥0.85. Garbage text → failed with populated lastError.

## Slice J — 24h file cleanup cron (parallel)

**Schema additions (small, additive — call out in Slice A's schema PR to avoid two pushes):**
- Add `fileExpiresAt: v.optional(v.number())` to `subscriptions`, `parseJobs`, `pendingParses`.
- Add index `by_fileExpiresAt` on each of those three tables.

**File to create:** `convex/cleanup.ts`
- `internalAction sweep({})` → walks each table via `internalQuery listExpired<Table>({ now, limit: 50 })` using the `by_fileExpiresAt` index. For each expired row: `internalMutation deleteFileAndClearRow({ tableName, rowId, fileId })` — `ctx.storage.delete(fileId)` (try/catch), patch row to clear `fileId` + `fileExpiresAt`.

**File to create:** `convex/crons.ts`
- `crons.interval("24h file cleanup", { hours: 1 }, internal.cleanup.sweep, {});`
- (Slice I appends its cron entry here.)

**Race-safety:** Set `fileExpiresAt` only when row's parse-or-edit lifecycle is complete:
- `parseJobs`: set at upload to `now + 25h` (extra hour buffer for parse).
- `subscriptions`: set when D's `confirm` mutation copies fileId from parseJob to subscription, value = `now + 24h`.
- `pendingParses`: set at insertion (when over-cap parse is queued), value = `now + 24h`.

**Verification:** Backdate a row's `fileExpiresAt` to past, invoke `internal.cleanup.sweep` from Convex dashboard → file gone, row's `fileId` cleared, F's "view receipt" disappears.

## Slice D — Parse review UI (sequential after B + C)

**File to create:** `app/dashboard/review/[jobId]/page.tsx`
- Subscribes to `parseJobs.getJob({ jobId })`. Renders by status:
  - `queued`/`running` → loading shimmer, copy "Reading your receipt…"
  - `succeeded` → editable `<SubForm>` pre-filled with `extracted.*`. If `isSubscription === false` OR `confidence < 0.6` → render inline note "Looks one-time — add anyway?" with Discard CTA.
  - `failed` → error card with `lastError` + "Try again" button.

**File to extend:** `convex/parseJobs.ts`
- `query getJob({ jobId })` → returns row + signed `fileUrl` if fileId set. Auth-checks.
- `mutation retry({ jobId })` → flips status to `queued`, reschedules `internal.parser.run`.
- `mutation confirm({ jobId, edited: extractedValidator })` →
  1. Auth + ownership check.
  2. Count subs: `.withIndex("by_user", q => q.eq("userId", userId)).take(11).then(r => r.length)`.
  3. If <10: insert `subscriptions` row (with `fileExpiresAt: Date.now() + 24h`), delete parseJob (NOT the storage file — its lifetime now belongs to the subscription). Return `{ kind: "added", subscriptionId }`.
  4. If ≥10: insert `pendingParses` row (`status: "awaiting_review"`, `fileExpiresAt: Date.now() + 24h`), delete parseJob. Return `{ kind: "queued_for_replacement", pendingId }`.

**Client routing after confirm:**
- `kind === "added"` → toast + `router.push("/dashboard")`.
- `kind === "queued_for_replacement"` → `router.push("/dashboard/replace/<pendingId>")`.

**Risks:**
- Discard race: if user discards while action mid-flight, `markSucceeded` fails on a deleted row. Wrap that mutation's `ctx.db.patch` in try/catch and treat NotFound as success.
- Date input timezone: client must format/parse in IST, not local. Use `lib/dates.ts:toStartOfDayIstMs`.

**Verification:** Confirm Netflix parse → subscriptions row appears, F's list shows it. Edit amount before confirm → stored value matches edit. Force a low-confidence parse → "looks one-time" affordance shown.

## Slice I — Renewal alert cron (sequential after F)

**File to create:** `convex/alerts.ts`
- `internalAction scanAndSend({})`:
  1. `tomorrowMs = startOfTomorrowIstMs(Date.now())`.
  2. Paginate `users` table 100 at a time via `internalQuery listAllUsers({ paginationOpts })`.
  3. Per user: `internalQuery findDueSubsForUser({ userId, dayMs: tomorrowMs })` using `by_user_and_nextRenewal` with `.eq("userId").eq("nextRenewal", dayMs)`.
  4. Per due sub: `internalMutation maybeSendOne({ userId, subscriptionId, dayMs })`.
- `internalMutation maybeSendOne`:
  1. Dedup: query `alerts` `by_subscription_and_type` for subId + type="day_before" with `sentAt >= dayMs`. If found → return.
  2. Insert `alerts` row with `sentAt: Date.now()` (idempotency lock).
  3. Schedule `internal.alerts.sendEmail({ alertId })`.
- `internalAction sendEmail({ alertId })` (`"use node";`):
  1. Load alert + subscription + user email.
  2. `await resend.sendEmail(ctx, { from: process.env.ALERTS_EMAIL_FROM!, to: user.email, ...emailTemplates.dayBefore(sub) })` where `resend` is the `@convex-dev/resend` component handle.

**File to create:** `convex/lib/emailTemplates.ts` — pure functions returning `{ subject, html, text }`. Hand-rolled HTML with inline styles (no JSX in actions; no custom fonts in email — use system stack).

**File to extend:** `convex/crons.ts`
- `crons.cron("renewal day-before alerts", "30 2 * * *", internal.alerts.scanAndSend, {});` (02:30 UTC = 08:00 IST)

**Risks:**
- TZ math: build a `convex/lib/dates.test.ts` (use convex-test + vitest per Convex guidelines) confirming `startOfTomorrowIstMs` matches stored sub `nextRenewal`.
- Cron miss: if Convex was down at 02:30 UTC, alerts skip that day. MVP accepts this.

**Verification:** Set a sub's `nextRenewal` to tomorrow IST start-of-day ms. Manually invoke `internal.alerts.scanAndSend` from Convex dashboard. Email arrives from `alerts@<domain>` within 30s. Re-invoke → no second email.

- **AI SDK v6 structured-output API.** `generateObject` is exported in v6 (verified in `node_modules/ai/dist/index.d.ts`). Some validator hooks may incorrectly flag it as removed — that's stale guidance.
- **Convex Auth Resend env var name.** Provider reads `AUTH_RESEND_KEY`. Typo → silent failure; no client-side error.
- **Next 16 middleware Proxy default.** Confirm Convex Auth's middleware helper is compatible; if not, opt back to Edge with `runtime = "edge"` in `middleware.ts`.
- **First push timing for Resend domain.** If verified domain isn't ready when A lands, set `AUTH_EMAIL_FROM` and `ALERTS_EMAIL_FROM` to `Renewl <onboarding@resend.dev>` for the first build, swap once verified.
- **Schema validator on push.** Wipe in §0.2 is mandatory or the swap rejects.
